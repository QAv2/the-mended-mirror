The Mended Mirror — the solo loop

A week of shard-production with aider + DeepSeek. One unit, repeated. Honest gold only.

The division of labor. You produce raw shards this week — the bulk, parallelizable half of the fan-out. The adversarial verify / tiering audit (the part that needs orchestration) waits for Claude. Nothing is wasted: verification is a later gate, not a write-time requirement. A Gemini-written shard is what a swarm agent produced — same spec, just human-paced.

▶ The easy way — the Forge (no terminal)

Launch it once, then it's all buttons in your browser:

./mend gui     # or: double-click "The Mended Mirror — Forge" in your app launcher

Each remaining shard is a row with a colour swatch and its anchor. Click ⚒ Forge → DeepSeek writes it (~30–60s) → you get figure count, seam count, and the tier spread right there. Then:

  • View on map ↗ — opens the live mirror so you can follow the new gold.
  • ✓ Keep & commit — merges, validates, commits. The row turns gold and the counter ticks.
  • ↻ Redo — don't like it? One click regenerates from scratch.

The Forge does make → validate → merge → sanity → commit behind every button. Periods auto-set; the old "no period" failure is gone. The terminal loop below still works if you ever want it — same engine.

§2The terminal loop (optional — the Forge does all this)

Three commands per shard. make calls DeepSeek directly (no aider for generation — aider's diff/whole edit machinery kept dropping fresh-file output; the direct API call is bulletproof and one command). Optionally ./mend brief <id> first to preview the brief, or ./mend queue 1 to see what's left.

  1. Generate the shard. DeepSeek writes data/_fanout/<id>.json, validates it's real JSON, and prints a summary (figure count, seam count, tier spread) with warnings if anything looks off (too few figures, missing facets, over-claimed tier-1s). 
    ./mend make judaism
# a stubborn/contested one? give it the reasoner:
./mend make judaism reasoner
  2. Build & validate. Merges your shard into the live dataset and runs the same sanity check the engine uses. 
    ./mend build
    ✓ clean → open the map and eyeball it.   ✗ problems → it tells you what's wrong (almost always a seam pointing at an id not in the inventory). Just ./mend make <id> again for a fresh draft, or hand-fix the JSON. Don't commit dirty.
    (Harmless: a no period for tradition warning — the timeline table doesn't know new traditions yet. Ignore it; Claude batch-fixes on return.)
  3. Eyeball the honesty — open index.html, find your new tradition, follow its gold seams. Are the convergences real? Are the tiers humble (mostly tier 3)? If yes: 
    git add -A && git commit -m "mirror: add judaism shard"

§3What "good" means (so the week isn't wasted)

CheckPass looks like
Figures7–11, each with a gloss (what it is) and a facet (why it can't be folded into another shard)
Gold seams8–20 cross-tradition edges, every b id a real one from the inventory
Tieringmostly tier 3 (analogous). Tier 1–2 only for proven descent/contact. Tier 4 must say "a loose parallel…" in the note
Sourcesprovenance names real attestation; contested/reconstructed material is flagged, not asserted
Sanity./mend build ends ✓ clean
The one unforgivable error is over-claiming. Bright fake gold poisons the whole thesis; faint honest gold is the point. When a shard reads as lazy perennialism ("everything is secretly the same"), reject it — tell aider to lower tiers and add facets. Flag, don't fold.

§4If something breaks while I'm gone

SymptomDo this
aider gives 402 / insufficient balanceYour DeepSeek credit ran dry — top up at platform.deepseek.com. (A week of shards should cost well under $1, so this shouldn't happen.)
aider gives 429 / rate-limitBrief pause then retry the same command — DeepSeek rate-limits are generous; this is transient.
./mend build shows ✗ dangling refA seam points to a non-existent id. In aider: "fix the seam whose b is X — seam to a joint id from the palette instead." Rebuild.
make says "invalid JSON" + saved a .RAW.txtDeepSeek emitted slightly-broken JSON (usually a trailing comma). Open the .RAW.txt to glance, or just re-run ./mend make <id> — a fresh draft almost always parses.
A shard came out badly / you want a redoJust ./mend make <id> again — it overwrites. The merge rebuilds from scratch every time, so it's clean.
A commit you dislikegit log --oneline then git revert <hash>, or git reset --hard HEAD~1.
Everything feels wedgedYou can always git reset --hard HEAD back to the last good commit. The baseline commit is your floor.
