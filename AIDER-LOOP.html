The Mended Mirror — the solo loop

A week of shard-production with aider + Gemini. One unit, repeated. Honest gold only.

The division of labor. You produce raw shards this week — the bulk, parallelizable half of the fan-out. The adversarial verify / tiering audit (the part that needs orchestration) waits for Claude. Nothing is wasted: verification is a later gate, not a write-time requirement. A Gemini-written shard is what a swarm agent produced — same spec, just human-paced.

§2The daily loop

  1. Pick the next shard and get its brief. It prints the exact aider command and the first message to paste — color, region, and anchor motif already filled in. 
    ./mend brief judaism
  2. Run the aider line it gave you. The spec and a finished shard ride along as read-only context; the new data/_fanout/<id>.json is the only file aider writes. Paste the first message, let it draft, /exit.
  3. Build & validate. This merges your shard into the live dataset and runs the same sanity check the engine uses. 
    ./mend build
    ✓ clean → open the map and eyeball it.   ✗ problems → it tells you what's wrong (almost always a seam pointing at an id that isn't in the inventory). Fix in aider, build again. Don't commit dirty.
  4. Eyeball the honesty — open index.html, find your new tradition, follow its gold seams. Are the convergences real? Are the tiers humble (mostly tier 3)? If yes: 
    git add -A && git commit -m "mirror: add judaism shard"

§3What "good" means (so the week isn't wasted)

CheckPass looks like
Figures7–11, each with a gloss (what it is) and a facet (why it can't be folded into another shard)
Gold seams8–20 cross-tradition edges, every b id a real one from the inventory
Tieringmostly tier 3 (analogous). Tier 1–2 only for proven descent/contact. Tier 4 must say "a loose parallel…" in the note
Sourcesprovenance names real attestation; contested/reconstructed material is flagged, not asserted
Sanity./mend build ends ✓ clean
The one unforgivable error is over-claiming. Bright fake gold poisons the whole thesis; faint honest gold is the point. When a shard reads as lazy perennialism ("everything is secretly the same"), reject it — tell aider to lower tiers and add facets. Flag, don't fold.

§4If something breaks while I'm gone

SymptomDo this
aider gives 429 / quota / rate-limitSwitch the model to gemini/gemini-2.5-flash in the command (or wait a minute). Flash has far higher free limits.
./mend build shows ✗ dangling refA seam points to a non-existent id. In aider: "fix the seam whose b is X — seam to a joint id from the palette instead." Rebuild.
A shard came out badly / you want a redogit checkout data/_fanout/<id>.json (or delete it), then re-brief. The merge is rebuilt from scratch every time, so deletions are clean.
aider edited the wrong file / a commit you dislikegit log --oneline then git revert <hash> — or /undo inside aider undoes its last commit.
Everything feels wedgedYou can always git reset --hard HEAD back to the last good commit. The baseline commit is your floor.
